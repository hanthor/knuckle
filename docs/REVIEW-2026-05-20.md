# Principal-Engineer Review — 2026-05-20

QA passes 3 and 4. Covers headless path, sysext pipeline, VM e2e automation,
and goroutine lifecycle.

## Findings — All Closed

### B4 — SSH keys not reaching Ignition (HIGH) — CLOSED

`detectLocalSSHKeys()` in `tui.go` only rendered a label string; it never
committed keys into the Model. Fixed: `StepUser` handler now calls
`detectLocalSSHKeys()` and stores keys in `m.cfg.Users[0].SSHKeys` before the
form is submitted.

### Sysexts silently dropped in headless path (HIGH) — CLOSED

`headless.Run()` passed `cfg.Sysexts []string` (names only) to
`ToInstallConfig()` without resolving them to `[]model.SysextEntry` (name +
URL + version). Fixed: `resolveSysexts(ctx, names, client)` added; fetches the
bakery catalog, builds a name→entry index, returns an error for any unknown
name. Wired as step 2b in `Run()`. Injectable `newBakeryClientFunc` var allows
tests to supply a `bakery.MockClient` without network access.

### Goroutine leak on force-quit during install (MEDIUM) — CLOSED

`startInstall()` in `tui.go` created a `context.WithCancel` but discarded the
`CancelFunc`. If the user force-quit with ctrl+c while `flatcar-install` was
running, the goroutine had no way to stop. Fixed: `installCancel
context.CancelFunc` added to Model; stored before the goroutine is launched;
called on all quit paths (`Update ctrl+c`, `handleKey ctrl+c`, `handleKey q`).

### qemu-img backing path doubled (LOW) — CLOSED

`just vm-e2e` produced `Could not open '.vm/.vm/flatcar_base.img'`. QEMU
resolves relative backing paths relative to the qcow2 file's directory, not
the working directory. Fixed: all three `qemu-img create` calls changed to use
`$(pwd)/.vm/flatcar_base.img` (absolute path).

### TOCTOU LOW in WriteIgnitionFile — ACKNOWLEDGED, NOT FIXED

`os.CreateTemp` + `chmod 0600` has a theoretical race on the permission set.
The kernel enforces `O_EXCL` atomically and the system temp dir is
mode-restricted; the window is not exploitable in practice. Documented in
`SECURITY.md` known gaps.

## VM e2e Automation

`just vm-e2e` now runs three passes automatically:

1. **DHCP** — verifies hostname, OEM update strategy, locksmith enabled.
2. **Static** — verifies `/etc/systemd/network/10-static.network` content.
3. **Sysext (docker)** — verifies `/etc/extensions/docker.raw` present and
   `docker version` exits 0 after boot.

Each pass uses a fresh overlay image. Timeouts: 15m (DHCP/static), 25m (sysext).
