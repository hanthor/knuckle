## Blockers (must-fix before any 1.0)

### B1 — Supply-chain claims oversold

**Location.** `internal/bakery/channels.go:84` and `internal/bakery/bakery.go`.

**Symptom.** README and AGENTS.md (pre-this-PR) advertised "SBOM/SHA512/GPG
verification" as Architecture Decision #8. Reality:

- The Flatcar **SBOM** SHA512 *is* verified against `.DIGESTS` and surfaced
  as `info.DigestVerified`. Good.
- The Flatcar **GPG signature** is only *presence-checked* — the `.asc` URL
  is GET'd and on HTTP 200 `info.SignedDigest = true`. No GPG validation
  ever happens.
- The **sysext bakery** downloads (`internal/bakery/bakery.go`) verify
  nothing. No SHA, no signature.

**Fix.** Two PRs:

1. Verify the `.DIGESTS.asc` signature against the Flatcar release key.
   Bundle the key as an embedded `[]byte`; rotate it with releases.
2. For each sysext, fetch the `.sha256` sidecar and validate the `.raw`
   bytes before parsing. Reject if absent.

### B2 — Reboot bypasses the runner abstraction

**Location.** `cmd/knuckle/main.go:174` and `internal/tui/tui.go:253`.

**Symptom.** Both reboot paths call `exec.Command("systemctl", "reboot")`
directly, violating the rule that all external commands route through
`internal/runner`. The TUI's `tea.ExecProcess` form is the correct Bubble
Tea pattern but still hardcodes the binary; the headless main path has no
`--dry-run` guard at the *exec* line (the surrounding `if !cfg.DryRun`
catches it for now, but the next refactor will trip on this).

**Fix.** Add `Reboot(ctx) error` to `runner.Runner`. `DryRunner` logs;
`RealRunner` shells `systemctl reboot`. TUI keeps `tea.ExecProcess` but
calls the runner method for the command construction.

### B3 — No pre-install disk-path validation in headless mode

**Location.** `internal/headless/headless.go` (`Run` and `ToInstallConfig`).

**Symptom.** `validate.DiskPath` exists in `internal/validate/validate.go`
but is never invoked on the headless config. The TUI is partially protected
via huh `.Validate()` callbacks, but headless config can pass any string —
including `../../etc/anything` — straight to `flatcar-install -d`.

**Fix.** Call `validate.DiskPath(cfg.Disk)` in `headless.Validate()` (which
runs before `Run`). Reject paths that don't start with `/dev/`.

## High (CNCF-incubating bar)

### H1 — No SBOM or signed release artifacts

**Location.** `.github/workflows/release.yml`.

**Recommendation.** Generate an SBOM with `syft` and sign the binary + ISO
with `cosign`. Publish both as release assets.

### H2 — No fuzz tests on input-parsing surfaces

**Location.** `internal/validate`, `internal/ignition`, `internal/bakery`.

**Recommendation.** Add `FuzzHostname`, `FuzzCIDR`, `FuzzSSHKey`,
`FuzzIgnitionURL`, `FuzzVersionTxt`. Run `-fuzztime=30s` in a nightly
GitHub Actions job (cron `0 7 * * *`).

### H3 — Coverage targets in TEST-PLAN.md aren't enforced

**Reality.** `internal/tui` 46% vs aspiration 70%. `internal/validate` 88%
vs aspiration 95%. `internal/install` 76% vs aspiration 80%.

**Recommendation.** The gates in `Justfile :: cover-check` are set
conservatively below current coverage. Ratchet them up issue-by-issue. Two
places that pay off fast:

- Refactor pure logic out of `internal/tui` into `internal/wizard` or
  `internal/validate` so it can be unit-tested without a fake terminal.
- Add table-driven tests for hostname (numeric-only, Unicode, all-digits)
  and timezone (`Etc/GMT+14`, path-traversal attempts) in `validate`.

### H4 — `just headless-test` is not in CI

**Symptom.** Headless mode shares ~75% of the install path with the TUI but
runs entirely without QEMU. It's cheap and high-signal. CI runs unit tests
only.

**Recommendation.** Add a `headless-e2e` job to `ci.yml`:

```yaml
- run: just headless-test
```

That single line walks the canned JSON config through `internal/headless`
end-to-end under `--dry-run`. Catches a class of headless regressions
unit tests miss.

### H5 — Goroutine lifetime in TUI install

**Location.** `internal/tui/tui.go` (install command spawn).

**Symptom.** A 30-minute `context.WithTimeout(context.Background(), ...)`
spawned for the install can outlive the TUI if the user double-Ctrl+C
during install — no shared cancel.

**Recommendation.** Hold the cancel in the model and call it from the
quitting branch of `Update`.
